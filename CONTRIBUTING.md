# Contributing to My Dotfiles

Thank you for your interest in contributing to this dotfiles repository! 

## How to Contribute

### Reporting Issues
- If you find a bug or have a suggestion, please open an issue
- Clearly describe the problem or enhancement
- Include your system information if relevant

### Submitting Changes
1. Fork the repository
2. Create a new branch for your feature (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test your changes thoroughly
5. Commit your changes (`git commit -m 'Add some amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Style Guidelines

### Config Files
- Use clear, descriptive comments
- Follow the existing style and formatting
- Keep configurations minimal and clean
- Document any non-obvious settings

### Shell Scripts
- Use bash for shell scripts
- Include error handling
- Add comments for complex operations
- Make scripts executable with `chmod +x`

## Adding New Configurations

When adding support for a new application:
1. Create a new directory under `.config/`
2. Add well-commented configuration files
3. Update the README.md with the new application
4. Update the installation script if needed
5. Test the configuration

## Questions?

Feel free to open an issue for any questions or discussions!
